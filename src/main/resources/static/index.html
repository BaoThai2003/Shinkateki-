Weakness Score Formula

weakness = (incorrect * 2)
         + (response_time / 1000)
         + (streak * 3)
0-3 Strong 4-8 Medium 8+ Weak

Every answer updates `performance_stats` and informs the next test generation.

II. Core Idea

Each answer becomes training data

Capture

`is_correct`, `response_time`, `attempt_count`, `streak`, `timestamp`.

→

Analyze

Compute weakness score using error weight + timing + streak.

→

Adapt

Prioritize weak characters in future tests and review plans.

III-IV. What Makes It Different

Behavior-driven adaptivity, not random repetition

Traditional Apps vs Shinkateki

Traditional App Shinkateki System
Incorrect → simple replay Incorrect → weakness analysis update
Random question order 70 / 20 / 10 weak-medium-strong distribution
No speed awareness Response-time reflex analysis
No psychology signal Tracks consecutive error sequences

Adaptive Test Generation

  • 70% Weak
  • 20% Medium
  • 10% Strong

Spaced Repetition

  • Very Weak → 5 minutes
  • Medium → 1 day
  • Strong → 3 days

State-Aware Logic

  • Consecutive errors > 5 reduces difficulty
  • Inject easier recovery questions
  • Avoid cognitive overload loops

VI. Database Design

Four focused tables for user learning intelligence

users

`id`, `username`, `password`, `level`, `total_score`

characters

`id`, `character`, `romaji`, `type` (hiragana / katakana)

attempts

`id`, `user_id`, `character_id`, `is_correct`, `response_time`, `created_at`

performance_stats

`user_id`, `character_id`, `weakness_score`, `last_reviewed`, `next_review`
